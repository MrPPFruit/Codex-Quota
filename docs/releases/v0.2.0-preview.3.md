# Codex Quota v0.2.0-preview.3

这是 Windows 11 x64 的第三个公开预览版，修复真实 Windows 反馈中的两个问题：官方桌面应用迁移为 ChatGPT 后额度会话没有启动，以及展开气泡的旋转色场在部分相位露出白色底层。macOS 版本不受影响。

## Windows 修复

- 同时识别 `Codex` 与 `ChatGPT` 进程名，但名称只作为发现提示；最终仍要求精确官方 Package Family Name、受保护 WindowsApps 包根、有效签名、相同证书指纹和 `app-server` 能力探针。
- 色场保持固定 160×160 DIP，不随 hover 缩放；Canvas 子层防止 WPF 父布局把旋转纹理压缩为展开窗口尺寸。
- Windows runner 对不可用额度状态覆盖 0/24/45/90/135° 色流相位与 96/144 DPI，验证左右两端和圆角内部没有露出无色底层。

## 下载与校验

下载：

- `Codex-Quota-v0.2.0-preview.3-windows-x64.zip`
- `Codex-Quota-v0.2.0-preview.3-windows-x64.zip.sha256`

本 Preview 没有 Authenticode 商业代码签名，SmartScreen 可能显示未知发布者。SHA-256 只能确认文件与本 Release 资产一致，不能替代开发者身份认证。请只从 `MrPPFruit/Codex-Quota` 官方 Release 下载，完整解压到固定目录后运行；不要直接从 ZIP 内运行。

## 预览边界

Windows runner 能验证编译、协议与 UI 回归、固定 WPF PNG、x64 PE、self-contained single-file、未签名状态、ZIP 内容与 SHA-256，但不能证明用户设备上的 Store helper 相对路径和真实额度。因此下载后仍需确认日志是否依次出现官方桌面包、可信 helper 与额度快照；若失败，请只提供 `%LOCALAPPDATA%\CodexQuota\Logs\current.log`，不要提供账户或认证文件。

统一版 ChatGPT 没有公开的“当前是否位于 Codex 模式”进程信号，因此本版以官方桌面宿主的启动/退出作为显示边界。安装、升级和完整验收步骤见 [Windows 分发说明](https://github.com/MrPPFruit/Codex-Quota/blob/main/docs/windows-distribution.md)。
